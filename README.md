A static website for the Dugout Devils cricket team, featuring live data fetched from [CricHeroes](https://cricheroes.com/team-profile/11061119/dugout-devils).

## 🚀 One-Time Setup (15 min)

### Step 1 — Push to GitHub

```bash
cd /Users/satyamkumar/Documents/Cricket/dugout_devils

git init
git add .
git commit -m "Initial commit — Dugout Devils website"

# Create a new repo on GitHub.com first, then:
git remote add origin https://github.com/<your-username>/dugout-devils.git
git branch -M main
git push -u origin main
```

### Step 2 — Enable GitHub Pages

1. Go to your repo on GitHub
2. **Settings → Pages → Source → GitHub Actions**
3. Save

> The workflow at `.github/workflows/fetch_and_deploy.yml` handles deployment automatically.

### Step 3 — Add Cookie Secrets (for automated data fetching)

1. On GitHub: **Settings → Secrets and variables → Actions → New repository secret**
2. Add two secrets:

| Secret Name | Value | Notes |
|---|---|---|
| `CRICHEROES_UDID` | `d366e93bfc229d57553a4cd0461e1656` | Stable, rarely changes |
| `CRICHEROES_CF_BM` | *(your current `__cf_bm` cookie)* | Expires ~monthly — update when needed |

**How to get fresh cookies:**
1. Open `https://cricheroes.com/team-profile/11061119/dugout-devils/matches` in Chrome
2. DevTools → Network → find the `matches.json` request
3. Right-click → **Copy → Copy as cURL**
4. Find `-b '...'` — paste each cookie value into the secrets above

### Step 4 — Trigger First Deploy

Go to **Actions → Fetch CricHeroes Data & Deploy → Run workflow**

## 🔄 Automated Updates

The GitHub Action runs **every 6 hours** (5:30 AM / 11:30 AM / 5:30 PM / 11:30 PM IST):

1. Runs `main.py` → fetches fresh CricHeroes data
2. Saves `dugout_devils_data.json` to the repo
3. Commits the updated JSON back
4. Deploys to GitHub Pages

The website's `fetchCricHeroesJSON()` function automatically loads the latest JSON on every page visit.

## 🛠 Manually Running the Fetcher


Output: `dugout_devils_data.json` — open `index.html` locally to see it load.

## 🍪 Cookie Expiry

- `udid` — very stable, update once every few months
- `__cf_bm` — Cloudflare bot token, may expire within days/weeks
- When the fetch fails (expired cookies), the **old JSON stays in place** — the site still works with the last-known data

**Signs cookies expired:** GitHub Action job fails with HTTP 403 → update `CRICHEROES_CF_BM` secret.
